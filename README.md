# Реалтермо — заявки

Сайт уже отправляет существующую форму в `POST /api/lead`.

Теперь сценарий такой:

1. Посетитель заполняет текущую форму на сайте.
2. Заявка сохраняется в локальную SQLite-базу.
3. Владелец открывает `/admin/leads`.
4. Входит по логину и паролю.
5. Видит список заявок и может отметить заявку как обработанную.

## Что настроить

Создайте `.env.local` на основе `.env.local.example`.

Нужные переменные:

- `ADMIN_USERNAME` — логин для входа в админку.
- `ADMIN_PASSWORD_HASH` — хеш пароля.
- `ADMIN_SESSION_SECRET` — случайная длинная строка для подписи сессии.
- `LEADS_DB_PATH` — необязательно. Если не задано, база лежит в `storage/realtermo_leads.sqlite`.

## Как создать хеш пароля

В проекте есть готовая команда:

```bash
npm run hash:admin-password -- "ВашСильныйПароль"
```

Скопируйте результат в `ADMIN_PASSWORD_HASH`.

Для `ADMIN_SESSION_SECRET` можно использовать длинную случайную строку, например 32+ символа.

## Где смотреть заявки

Откройте:

```text
/admin/leads
```

Пример для продакшена:

```text
https://ваш-домен/admin/leads
```

## Как войти

1. Откройте `/admin/leads`.
2. Введите `ADMIN_USERNAME`.
3. Введите пароль, для которого был создан `ADMIN_PASSWORD_HASH`.

## Как поменять пароль

1. Выполните:

```bash
npm run hash:admin-password -- "НовыйПароль"
```

2. Замените значение `ADMIN_PASSWORD_HASH` в `.env.local` или в переменных окружения на хостинге.
3. Перезапустите приложение.

## Где лежит база

По умолчанию:

```text
storage/realtermo_leads.sqlite
```

Папка `storage/` не является публичной частью Next.js-приложения. Дополнительно в ней лежит `.htaccess` с запретом прямого доступа.

## Как сделать резервную копию

Достаточно скопировать файл базы:

```text
storage/realtermo_leads.sqlite
```

Например:

1. Скачать файл с хостинга через файловый менеджер/FTP.
2. Или сделать копию рядом с другим именем.

## Как открыть на телефоне и закрепить

Открытие:

1. Откройте в браузере `https://ваш-домен/admin/leads`.
2. Войдите под своим логином и паролем.

Закрепление:

1. На компьютере: откройте страницу и добавьте ее в закладки или создайте ярлык браузером.
2. На iPhone: Safari → Поделиться → На экран «Домой».
3. На Android: Chrome → Меню → Добавить на главный экран.

## Что уже реализовано

- Сохранение заявок в SQLite.
- Проверка обязательного телефона через существующую валидацию формы.
- Honeypot-поле против простых ботов.
- Защита от дублей по тому же телефону за последние 5 минут.
- Закрытая страница просмотра заявок.
- Кликабельные телефоны через `tel:`.
- Статусы `new` / `processed`.

## Проверка перед запуском

1. Заполнить `.env.local`.
2. Запустить `npm run dev`.
3. Отправить тестовую заявку с сайта.
4. Открыть `/admin/leads`.
5. Проверить вход.
6. Проверить, что новая заявка появилась сверху.
7. Нажать `Обработано` и убедиться, что статус изменился.
